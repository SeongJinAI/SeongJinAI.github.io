# LinkedIn Post — Hotfix Pipeline

> 대상: 개발자/CTO/테크리드
> 분량: ~1,250자
> 첨부: 다이어그램 이미지

---

서버 에러가 발생하면 매번 반복되는 루틴이 있습니다.

알림 확인 → SSH 접속 → 로그 분석 → 원인 파악 → 수정 → PR → 리뷰 → 배포

에러 하나에 30분~수 시간. 그런데 절반 이상이 DB 스키마 불일치처럼 코드 변경이 필요 없는 단순 이슈입니다.

이 문제를 해결하기 위해 AI 기반 자동 핫픽스 파이프라인을 구축했습니다.

—

핵심 설계: 에러 유형 자동 라우팅

모든 에러를 동일하게 처리하지 않습니다.
AI가 스택트레이스를 분석해 에러를 3가지로 자동 분류합니다.

🔴 DB 에러 → ALTER SQL을 Slack으로 직접 전달
🟡 코드 에러 → AI가 코드 수정 후 PR 자동 생성
🟠 혼합 → SQL + PR 동시 처리

DB 에러에 코드를 뒤지는 시간이 사라집니다.

—

도메인 특화 서브에이전트

단일 AI가 전체 시스템을 분석하면 컨텍스트 낭비와 오진단이 발생합니다.
이 시스템은 도메인별 전문 서브에이전트가 해당 영역의 Entity 관계, 비즈니스 규칙, 에러 패턴을 기반으로 정밀 분석합니다.

—

SOTT 원칙: AI가 분석하고, 사람이 결정한다

AI에게 무제한 권한을 주지 않습니다.
"이 에러를, 이 범위에서만, 이번 한 번만" — Scoped One-Time Task.

개발자 개입은 딱 3번:
1️⃣ 수정 계획 승인 (1분)
2️⃣ DB SQL 실행 (1분)
3️⃣ PR 최종 머지 (2분)

나머지는 전부 자동입니다.

—

결과:
• DB 에러: 30분 → 2분 (93% 단축)
• 코드 에러: 1-2시간 → 10분 (83-91% 단축)
• 야간/주말: 즉시 감지 + 분석 (익일 대응 → 실시간)

기술 스택: Sentry + Claude Code + Slack + GitHub CLI + Scheduled Trigger
